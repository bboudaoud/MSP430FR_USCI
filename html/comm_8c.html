void confUCB0 ( unsigned int  commID)
inline

Configures USCI B0 for operation.

Write control registers and clear system variables for the USCI B0 module, which can be used as either a UART or SPI.

NOTE: This config function is already called before any read/write function call and therefore should (in almost all cases) never be called by the user.

Parameters
commIDThe communication ID for the registered app

int registerComm ( usciConfig *  conf)

Registers an application for use of a USCI module.

Create a USCI "socket" by affiliating a unique comm ID with an endpoint configuration for TI's eUSCI module.

conf The USCI configuration structure to be used (see comm.h) commID A positive (> 0) value representing the registered app -1 The maximum number of apps (MAX_DEVS) has been registered

int spiB0Read ( unsigned int  len,
unsigned int  commID 
)

Receive method for USCI B0 SPI operation.

This method performs a synchronous read by storing the bytes to be read in spiB0RxSize and then performing len dummy write to the bus to fetch the data from a slave device. The RX size is cleared on this function call.

Parameters
lenThe number of bytes to be read from the bus
commIDCommunication ID number of the application
Return values
-1USCI B0 Module Busy
1Receive successfully started
Side Effects:
Reset the UCB0 RX size and data pointer

int spiB0Write ( unsigned char *  data,
unsigned int  len,
unsigned int  commID 
)

Transmit method for USCI B0 SPI operation.

This method initializes a transmission of len bytes from the base of the *data pointer. Similarly to uartB0Write(), the transmission uses the USCI B0 TX ISR to complete, so 2 sequential calls may result in partial transmission.

Parameters
*dataPointer to data to be written
lenLength (in bytes) of data to be written
commIDCommunication ID number of application
Return values
-1USCI B0 Module busy
1Transmit successfully started

__interrupt void usciB0Isr ( void  )

USCI B0 RX/TX Interrupt Service Routine.

This ISR manages all TX/RX proceedures with the exception of transfer initialization. Once a transfer (read or write) is underway, this method assures the correct amount of bytes are written to the correct location.
